Hexo

Hexo

Hexo是一个快速、简洁且高效的博客框架。Hexo使用Markdown(或其他渲染引擎)解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

1 安装Hexo

1.1 安装Git

参考安装Git。

1.2 安装Node.js

Node.js版本需不低于10.13，建议使用12.0及以上版本。

  • 进入官网地址下载长期维护本版安装包即可，此处下载64版本的.msi安装包；

  • 双击文件安装，.msi安装包安装完成后回自动添加环境变量；

  • 打开命令行工具，执行命令node -v可查看Node.js版本；

  • 安装包中包含Node官方包管理工具(Node Package Manager, npm)，执行命令npm -v可查看版本。

1.3 Github Pages部署博客

如果选择在Github Pages使用Hexo部署博客，需要有一个Github账号。

  • 创建一个仓库用于部署博客

    • 进入仓库(Repositories)页面；
    • 点击New新建一个仓库；
    • 仓库名格式需为<用户名>.github.io；
    • 权限设置为Public；
    • 点击Create repositorie创建仓库。

  • 创建博客主页

    • 点击creating a new file创建一个文件，文件名必须为index.html；
    • 随意写些内容，如下： 
      <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>delugewalker</title>
</head>
<body>
    <h1>delugewalker的个人主页</h1>
    <h1>Hello World!</h1>
</body>
</html>
    • 点击Commit new file提交。

  • 访问博客主页

    • 进入仓库 -> Settings -> Pages
    • 如果显示 Your site is published at https://delugewalker.github.io/，即可访问主页。

2 Hexo主题

Fluid主题 使用自定义字体

本文介绍如何在Hexo的Fluid主题中设置自定义字体。通常在主题对应的_config.yml文件中可以快速的设置常用字体，当访问站点的设备已安装该字体时，则可以在页面加载时正确显示。如果选择的字体在设备上未安装，则需要在CSS样式中嵌入该字体，通常是使用@font-face属性完成嵌入。

建立文件目录

  • 在Hexo项目根路径(非FLuid主题路径)找到文件夹source/fonts，若不存在则创建；
  • 在Hexo项目根路径下的主题样式目录themes/fluid/source/css中找到自定义样式文件custom.css，若不存在则创建；
  • 在Fluid主题对应的_config.yml文件中找到custom_css，添加上述自定义样式文件/css/custom.css。

下载字体

常见字体推荐在自助式Google字体网站获取，例如思源宋体(Noto Serif SC)。

  1. 选择字符集
    默认情况应用到西文(latin)，根据需要应用到其他字符集。此处同时勾选中文简体和西文。

  2. 选择字体样式
    根据自身情况选择字体粗细。

  3. 复制@font-face代码
    将@font-face代码写入/css/custom.css中。

  4. 下载字体文件压缩包
    将文件解压后放入先前建立的字体目录中。

  5. 指定Fluid主题_config.yml文件中的字体font-family: Noto Serif SC

对于上述网站未收录字体，例如汉仪唐美人(HANYI TangMeiRen)，可以去相应的字体网站下载，然后仿照上述@font-face代码填入字体路径和参数。
