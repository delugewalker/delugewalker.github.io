Hexo

Hexo

Hexo是一个快速、简洁且高效的博客框架。Hexo使用Markdown(或其他渲染引擎)解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

1 安装Hexo

1.1 安装Git

参考安装Git。

1.2 安装Node.js

Node.js版本需不低于10.13，建议使用12.0及以上版本。

  • 进入官网地址下载长期维护本版安装包即可，此处下载64版本的.msi安装包；

  • 双击文件安装，.msi安装包安装完成后回自动添加环境变量；

  • 打开命令行工具，执行命令node -v可查看Node.js版本；

  • 安装包中包含Node官方包管理工具(Node Package Manager, npm)，执行命令npm -v可查看版本。

1.3 Github Pages部署博客

如果选择在Github Pages使用Hexo部署博客，需要有一个Github账号。

  • 创建一个仓库用于部署博客

    • 进入仓库(Repositories)页面；
    • 点击New新建一个仓库；
    • 仓库名格式需为<用户名>.github.io；
    • 权限设置为Public；
    • 点击Create repositorie创建仓库。

  • 创建博客主页

    • 点击creating a new file创建一个文件，文件名必须为index.html；
    • 随意写些内容，如下： 
      <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>delugewalker</title>
</head>
<body>
    <h1>delugewalker的个人主页</h1>
    <h1>Hello World!</h1>
</body>
</html>
    • 点击Commit new file提交。

  • 访问博客主页

    • 进入仓库 -> Settings -> Pages
    • 如果显示 Your site is published at https://delugewalker.github.io/，即可访问主页。

1.4 创建Hexo项目

  • 创建一个Hexo项目并初始化

    $ hexo init hexo-blog
$ cd hexo-blog
$ npm install

  • 在本地启动Hexo

    $ hexo g
$ hexo s

  • 浏览器访问http://localhost:4000，页面默认主图风格如下：

2 Hexo配置与命令

2.1 配置文件

在Hexo博客项目的根路径存在一个_config.yml文件，在该文件中可以修改大部分的配置。

  • title：网站标题；
  • subtitle：网站副标题；
  • description：网站描述；
  • keywords：网站的关键词，支持多个关键词；
  • author：您的名字；
  • language：网站使用的语言，对于简体中文用户来说可设置为zh-CN；
  • timezone：网站时区，Hexo默认使用电脑的时区；
  • …

2.2 常用命令

2.2.1 init

语法：

$ hexo init [folder]

新建一个网站。如果没有设置folder，Hexo默认在目前的文件夹建立网站。

本命令相当于执行了以下几步：

  • git clone hexo-starter并git hexo-theme-landscape主题到当前目录或指定目录；
  • npm等包管理器下载依赖。

2.2.2 new

语法：

$ hexo new [layout] <title>

新建一篇文章。如果没有设置layout的话，默认使用_config.yml文件中的default_layout参数代替。如果标题包含空格的话，请使用引号括起来。

$ hexo new "post title with whitespace"

常用Options：

  • -p, --path：自定义新文章的路径
  • -r, --replace：如果存在同名文章，将其替换
  • -s, --slug：文章的Slug，作为新文章的文件名和发布后的URL

默认情况下，Hexo会使用文章的标题来决定文章文件的路径。对于独立页面来说，Hexo会创建一个以标题为名字的目录，并在目录中放置一个index.md文件。你可以使用--path参数来覆盖上述行为、自行决定文件的目录：

$ hexo new page --path about/me "About me"

2.2.3 generate

语法：

$ hexo generate

或简写为：

$ hexo g

常用Options：

  • -d, --deploy：文件生成后立即部署网站
  • -w, --watch：监视文件变动
  • -b, --bail：生成过程中如果发生任何未处理的异常则抛出异常
  • -f, --force：强制重新生成文件，Hexo引入了差分机制，如果public目录存在，那么hexo g只会重新生成改动的文件，使用该参数的效果接近hexo clean && hexo generate
  • -c, --concurrency：最大同时生成文件的数量，默认无限制

2.2.4 publish

语法：

$ hexo publish [layout] <filename>

发表草稿。

2.2.5 server

语法：

$ hexo server

或简写为：

$ hexo s

启动服务器。默认情况下访问网址为：http://localhost:4000/。

常用Options：

  • -p, --port：重设端口
  • -s, --static：只使用静态文件
  • -l, --log：启动日记记录，使用覆盖记录格式

2.2.6 deploy

语法：

$ hexo deploy

或简写为：

$ hexo p

用于将内容部署到网站。

常用Options：

  • -g, --generate：部署之前预先生成静态文件

2.2.7 render

语法：

$ hexo render <file1> [file2] ...

渲染文件。

常用Options：

  • -o, --output：设置输出路径

2.2.8 migrate

语法：

$ hexo migrate <type>

从其他博客系统迁移内容。

2.2.9 clean

语法：

$ hexo clean

清除缓存文件(db.json)和已生成的静态文件(public)。在某些情况(尤其是更换主题后)，如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

2.2.10 list

语法：

$ hexo list <type>

列出网站资料。

2.2.11 version

语法：

$ hexo version

显示Hexo版本。

2.2.12 选项

  • 安全模式

    $ hexo --safe

    在安全模式下，不会载入插件和脚本。当您在安装新插件遭遇问题时，可以尝试以安全模式重新执行。

  • 调试模式

    $ hexo --debug

    在终端中显示调试信息并记录到debug.log。

  • 简洁模式

    $ hexo --silent

    隐藏终端信息。

  • 自定义配置文件的路径

    # 使用 custom.yml 代替默认的 _config.yml
$ hexo server --config custom.yml

# 使用 custom.yml 和 custom2.json，其中 custom2.json 优先级更高
$ hexo generate --config custom.yml,custom2.json,custom3.yml

    自定义配置文件的路径，指定这个参数后将不再使用默认的_config.yml。你可以使用一个YAML或JSON文件的路径，也可以使用逗号分隔(无空格)的多个YAML或JSON文件的路径。例如：

    # 使用 custom.yml 代替默认的 _config.yml
$ hexo server --config custom.yml

# 使用 custom.yml, custom2.json 和 custom3.yml，其中 custom3.yml 优先级最高，其次是 custom2.json
$ hexo generate --config custom.yml,custom2.json,custom3.yml

    当你指定了多个配置文件以后，Hexo会按顺序将这部分配置文件合并成一个_multiconfig.yml。如果遇到重复的配置，排在后面的文件的配置会覆盖排在前面的文件的配置。这个原则适用于任意数量、任意深度的YAML和JSON文件。

  • 显示草稿

    $ hexo --draft

    显示source/_drafts文件夹中的草稿文章。

  • 自定义工作目录

    $ hexo --cwd /path/to/cwd

    自定义当前工作目录(Current working directory)的路径。

3 Hexo主题

Hexo官方提供了很多主题，可以根据自身喜好更改，常见的两种主题为Fluid和NexT，此处选择Fluid作为模板讲解主题的使用方法。

3.1 更换主题

更换方式如下：

  • 安装主题
    git clone以上主题的代码仓库到博客根路径下的themes目录内，也可下载最新的release版本，解压到该文件夹，可以修改下载的主题文件夹名称为’fluid’或’next’等。
  • 指定主题
    修改Hexo博客目录中的_config.yml配置项，例如theme:fluid。
  • 创建关于页(About)
    首次使用主题的关于页需要手动创建： 
    hexo new page about
    创建成功后，编辑博客目录下source/about/index.md，添加layout属性。修改后的文件示例如下： 
    ---
title: about
date: 2021-05-01 19:20:33
layout: about
---

<!-- 这里写关于页的正文，支持Markdown，HTML -->

对于Fluid主题用户，可阅读官方文档，通过配置指南可以解决绝大部分问题。

4 常用操作

4.1 插入图片

除了官方文档中建议的图片引用方式，此处推荐使用HTML标签<div>和<img>实现图片的插入，通过修改这两个标签的style可以实现对图片样式的自定义。

例如：

<div style="text-align: center">
<img style="margin: 5px 5px; border-radius: 0.3125em; box-shadow: 0 2px 4px 0 rgba(34,36,38,0.12), 0 2px 10px 0 rgba(34,36,38,0.08);" 
src="wallpaper.jpg" 
alt="插入图片" 
width="60%">
</div>

4.2 插入Latex公式

Hexo5.0以上，可尝试Hexo官方的hexo-math插件。

主题内Latex设置可参照Fluid文档设置，Katex速度更快，Mathjax覆盖更全面，本文偏向于使用Katex。

需要注意的是，在公式中不能(实际上文章中都不能)出现{ {(连续的大括号)，需用空格隔开，否则可能会出现渲染错误。

4.3 更换代码块风格

4.3.1 更换代码块背景色

Fluid主题配置文件_config.fluid.yml(此处使用根目录下的覆盖配置)中未给出代码块背景色的快速修改选项，如果想要更改代码块背景色可以修改主题的样式文件themes\fluid\source\css\_variables\base.styl中的highlight-bg-color参数。

可以直接在该样式文件内直接修改，也可以像其他颜色参数一样写入主题配置文件中。
例如在样式文件base.styl中：

$highlight-bg-color = theme-config("color.highlight_bg_color", "#f6f8fa")
$highlight-bg-color-dark = theme-config("color.highlight_bg_color_dark", "#2d333b")

并在主题配置文件_config.fluid.yml中写入：

color:
  # 代码块背景色
  highlight_bg_color: "#f6f8fa"
  highlight_bg_color_dark: "#496766"

4.3.2 更换代码块风格

在主题路径下使用自定义样式文件themes\fluid\source\css\custom.css，没有则自行创建，在主题配置文件_config.fluid.yml中将自定义样式文件指定为该文件custom_css: /css/custom.css。

在custom.css文件中写入以下代码可以实现Mac风格的代码块：

.highlight {
	 background: #011627;
	 border-radius: 5px;
	 box-shadow: 0 10px 30px 0 rgba(0,0,0,0.4);
	 padding-top: 30px;
}
 .highlight::before {
	 background: #fc625d;
	 border-radius: 50%;
	 box-shadow: 20px 0 #fdbc40, 40px 0 #35cd4b;
	 content: ' ';
	 height: 12px;
	 left: 12px;
	 margin-top: -20px;
	 position: absolute;
	 width: 12px;
}

代码块样式也可以根据自身喜好自由发挥。

4.4 使用自定义字体

本文介绍如何在Hexo的Fluid主题中设置自定义字体，其他主题方式类似。通常在主题对应的_config.yml文件中可以快速的设置常用字体，当访问站点的设备已安装该字体时，则可以在页面加载时正确显示。如果选择的字体在设备上未安装，则需要在CSS样式中嵌入该字体，通常是使用@font-face属性完成嵌入。

4.4.1 建立文件目录

  • 在Hexo项目根路径(非FLuid主题路径)找到文件夹source/fonts，若不存在则创建；
  • 在Hexo项目根路径下的主题样式目录themes/fluid/source/css中找到自定义样式文件custom.css，若不存在则创建；
  • 在Fluid主题对应的_config.yml文件中找到custom_css，添加上述自定义样式文件/css/custom.css。

4.4.2 下载字体

常见字体推荐在自助式Google字体网站获取，例如思源宋体(Noto Serif SC)。

  1. 选择字符集
    默认情况应用到西文(latin)，根据需要应用到其他字符集。此处同时勾选中文简体和西文。

  2. 选择字体样式
    根据自身情况选择字体粗细。

  3. 复制@font-face代码
    将@font-face代码写入/css/custom.css中。

  4. 下载字体文件压缩包
    将文件解压后放入先前建立的字体目录中。

  5. 指定Fluid主题_config.yml文件中的字体font-family: Noto Serif SC

对于上述网站未收录字体，例如汉仪唐美人(HANYI TangMeiRen)，可以去相应的字体网站下载，然后仿照上述@font-face代码填入字体路径和参数。
